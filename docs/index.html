Why this matters

collaboration semantics
  • Exists now: role taxonomy distinguishes HYPOTHESIS vs GROUND_TRUTH.
  • Exists now: provenance is a transformation chain, not just a source string.
  • Exists now: dependencies can point at cells, columns, external resources, or composites.
  • Missing in Forge today: ForgeArtifact, ForgeExecutionResult, and CollaborationEvent do not carry CellOverlay / BlackboardContext.
  • Missing lowering: no default projection from workflow outputs into cursor cells with overlay metadata.

Honest gap: Forge sync currently uses MutableSharedFlow. PRELOAD-native sync wants a workspace element that owns lifecycle and fanout, with cursor mutation rows as the event body.

Current → target mapping

what to lower next
  • Current: CollaborationEvent.CursorPosition already names the right presence primitive.
  • Current: sync is row-agnostic patches, not cursor mutation batches.
  • Target: ForgeWorkspaceElement : AsyncContextElement
  • Target: fanout subscribers = UI / audit / artifact indexer / remote sync
  • Target: event payloads should be cursor rows or mutation rows, not only ad hoc event classes

Missing PRELOAD lowering for provenance

from gap analysis
  • Need: ForgeArtifactCursor rows with dependency handles back to source rows and run events.
  • Need: content-addressed bodyRef / ConfixDoc split, not only embedded strings.
