## 1. Introduction

This document establishes a **formally complete, mutually exclusive taxonomy** of *use signalling primitives* for the TrikeShed framework. A *use signalling primitive* is the atomic unit of user-facing observable and actionable state in TrikeShed's LCNC (Logic / Computation / Notification / Coupling) architecture.

### 1.1 Core Thesis

> **A use signalling primitive is a triple**  
> **`Use × Carrier × Seam`**

where:
- **Use** ∈ `{Observe, Subscribe, Command, Query}` — the *intent* of the interaction
- **Carrier** ∈ the free algebra generated by `{Finite, Ordered, Sequence, Product, Tensor}` — the *shape* of the value
- **Seam** ∈ `{Logic, Computation, Notification, Coupling}` — the *execution tier* that processes the primitive

This factorization is **canonical**: every user-facing signal in TrikeShed decomposes uniquely into one element of each factor.

## 2. The Use Factor

### 2.1 Definitions

| Primitive | Formal Definition | TrikeShed Witness |
|-----------|-------------------|-------------------|
| **Observe** | `Observe : ∀A. Signal<A> → A` — read current value without subscription | `Signal.value`, `SignalSource.emit` |
| **Subscribe** | `Subscribe : ∀A. Signal<A> → Flow<A>` — reactive stream of future values | `Signal.changes`, `Flow` subscription |
| **Command** | `Command : ∀A. (A → A) × SignalSource<A> → Unit` — fire-and-forget mutation | `SignalSource.emit`, `toggle()`, `setValue()` |
| **Query** | `Query : ∀A B. (A → B) × Signal<A> → B` — correlated request/response | `Signal.map`, `Signal.combine`, `SignalAlgebra` composition |

### 2.2 Irreducibility of Query

**Theorem**: `Query` is not reducible to `Command × Subscribe`.

*Proof sketch*: A `Command × Subscribe` pair yields two independent channels — a write channel and a read channel — with no intrinsic correlation between the command issued and the value observed. `Query` requires a *single* correlation site: the responder binds the request parameters to the response value *atomically*. In TrikeShed this is witnessed by `Signal.map` / `Signal.combine` / `SignalAlgebra.beside` where the transformation is *pure* and the result is a *derived* signal whose value is definitionally equal to `f(source.value)` at every instant. No subscription race or command/response mismatch can occur. ∎

### 2.3 Subsumption Note

`Observe` is a degenerate `Query` with identity transformation. We keep it distinct because TrikShed's LCNC mode routing treats *read-only* access (Logic seam) differently from *derived* access (Computation seam).

## 3. The Carrier Factor

### 3.1 Primitive Carrier Constructors

The carrier algebra is the **free symmetric monoidal category** generated by five base objects:

| Constructor | Kind | Arity | TrikeShed Instance |
|-------------|------|-------|-------------------|
| **Finite** | `FinSet` | 0 | `Boolean`, `Enum` values — `Toggle`, `IdiotLight`, `MomentaryButton`, `RadioToggle<T>` |
| **Ordered** | `ℝ-interval` | 0 | `Double` range — `Slider`, `Knob`, `LevelMeter` |
| **Sequence** | `List<A>` | 1 | Discrete positions — `Dial<T>`, `TextField` edit sequence |
| **Product** | `A × B` | 2 | `TextFieldState = String × Int × Int × Int × Bool × Bool` |
| **Tensor** | `A ⊗ B` | 2 | `SignalAlgebra.beside/above/overlay : Component<A> × Component<B> → Component<A ⊗ B>` |

> **Notation**: `A ⊗ B` denotes the *tensor* (parallel composition) carrier; `A × B` denotes the *product* (structural tuple) carrier.

### 3.2 Carrier Normal Form

Every carrier expression reduces to a **unique normal form**:

```
Carrier ≃ (Finite | Ordered | Sequence) ⊗* (Product* (Finite | Ordered | Sequence))
```

**Reduction rules**:
1. `Product` and `Tensor` associate to the right
2. `Product` distributes over `Tensor` only when the left operand is a base carrier
3. All `Finite/Ordered/Sequence` primes appear *before* any `Product/Tensor` combinator

**Examples**:
- `TextFieldState` = `String × Int × Int × Int × Bool × Bool`  
  → `Product(Product(Product(Product(Product(Sequence(Char), Finite(Int)), Finite(Int)), Finite(Int)), Finite(Bool)), Finite(Bool))`  
  → NF: `Sequence ⊗ Product(Finite⁵)` (text is a character sequence; the five metadata fields are finite)
- `SignalComponent<Pair<A,B>>` from `beside` = `A ⊗ B` (tensor, not product)
- `RadioToggle<T>` = `Finite(T)` (single finite choice from options)

### 3.3 Mutual Exclusivity of Carrier Constructors

**Theorem**: The five constructors generate disjoint carrier classes.

*Proof*:
- `Finite` carriers have cardinality `< ℵ₀` and no intrinsic order/metric
- `Ordered` carriers embed in `ℝ` with dense order and metric
- `Sequence` carriers are free monoids on a base carrier — have concatenation and empty element
- `Product` carriers are tuples with fixed arity and projection maps
- `Tensor` carriers are parallel compositions with no projections (only `SignalAlgebra` combinators)

No carrier can inhabit two classes because the witnessing operations (`cardinality`, `order`, `concat`, `proj₁`, `beside`) are mutually exclusive on the same value. ∎

## 4. The Seam Factor

### 4.1 LCNC Mode Definitions

From `xvm/lib_cursor/.../FacetedCursorContract.kt`:

| Seam | Constant | Responsibility | Trigger |
|------|----------|----------------|---------|
| **Logic** | `LCNCMode.LOGIC = 0` | Pure transforms, no side effects, deterministic | `FieldSynapse.L_GET / L_SET` (logical) |
| **Computation** | `LCNCMode.COMPUTATION = 1` | CPU-bound vectorized ops, DuckDB engine | Derived signals, `Signal.map`, `Signal.combine` |
| **Notification** | `LCNCMode.NOTIFICATION = 2` | Fanout to subscribers, CCEK lifecycle | `Signal.changes` emission, `SignalSource.emit` |
| **Coupling** | `LCNCMode.COUPLING = 3` | ioctl → kernel boundary, btrfs execution | `FieldSynapse.P_GET / P_SET` (physical) |

### 4.2 Seam Routing Rules

| Use | Primary Seam | Notes |
|-----|--------------|-------|
| `Observe` | Logic | Direct value read; no derivation |
| `Subscribe` | Notification | Always lands here — fanout is the notification seam's defining duty |
| `Command` | Coupling (if physical) / Computation (if derived) | `emit` on physical extent → Coupling; `emit` on logical signal → Computation |
| `Query` | Computation | All derived signals route here; `SignalAlgebra` composition is pure Computation |

> **Critical invariant**: `Subscribe` *always* routes through Notification seam, even when the producing composite originates in Logic or Computation. The fanout mechanism is the seam boundary.

## 5. Primitive Constructors (Minimal Generating Set)

The complete taxonomy is generated by the following **minimal primitive constructors**:

```kotlin
// Use constructors (4)
Observe  : ∀A. Signal<A> → A
Subscribe : ∀A. Signal<A> → Flow<A>
Command   : ∀A. (A → A) × SignalSource<A> → Unit
Query     : ∀A B. (A → B) × Signal<A> → B

// Carrier constructors (5)
Finite    : Set<A> → Carrier          // |A| < ℵ₀
Ordered   : Interval<ℝ> → Carrier     // [min, max] × step?
Sequence  : Carrier → Carrier         // Free monoid (List)
Product   : Carrier × Carrier → Carrier  // Tuple
Tensor    : Carrier × Carrier → Carrier  // Parallel composition

// Seam constructors (4)
Logic       : Seam
Computation : Seam
Notification: Seam
Coupling    : Seam
```

**Total primitives**: 13 constructors generate the entire taxonomy.

## 7. Completeness Proof

### 7.1 Statement

> **Theorem**: The taxonomy `Use × Carrier × Seam` with constructors from §5 covers *every* user signalling primitive in the TrikeShed framework.

### 7.2 Proof Structure

We prove completeness by **exhaustive case analysis** on the TrikeShed codebase surface area.

#### 7.2.1 Window Toolkit (libs/window-toolkit, libs/user-signals)

All 10 signal interfaces (`Toggle` through `TextField`) and 4 `SignalAlgebra` combinators are classified in §6. The base algebra (`Signal<T>`, `SignalSource<T>`) provides exactly the `Observe/Subscribe/Command` interface; `Query` emerges from the `map/combine` methods and `SignalAlgebra` composition. No other public interfaces exist.

#### 7.2.2 LCNC Cursor Layer (xvm/lib_cursor)

The `FacetedCursorContract` defines exactly 4 LCNC modes (Logic, Computation, Notification, Coupling) and 4 FieldSynapse opcodes (L_GET, L_SET, P_GET, P_SET). The `FacetEngine.dispatch` matches each opcode to exactly one mode. All cursor transforms (`withPointcut`, `inMode`, `tagged`, `eval`) are classified. The carrier `Cell = Join<Any?, ColumnMetaRef>` and `CellCursor = Series<Join<Cell, SlabFacet>>` are built from `Product` and `Sequence` constructors. No other public API exists.

#### 7.2.3 ActiveJS / Expect Layer (libs/activejs)

The `ConfixRole` enum has exactly three values: `OBSERVATION`, `COMMAND`, `QUERY`. These map bijectively to `Observe`, `Command`, `Query`. `Subscribe` is implicit in the `Series` / `Cursor` reactive model. The `LiveQuery` carrier is a filter specification over `CoordinateRow`; its result is a `Series<CoordinateRow>` — a `Sequence(Product(...))` carrier. No other roles exist.

#### 7.2.4 Cross-Cutting: SignalSource.emit

`SignalSource.emit` / `emitSuspend` is the *only* mutation primitive in the codebase. It always pairs a `Command` use with a `Coupling` or `Computation` seam depending on whether the target carrier is physical (extent) or logical (derived). This is fully captured.

### 7.3 No Missing Primitives

Suppose a missing primitive `P` exists. Then `P` must:
1. Inhabit a `Use` not in `{O,S,C,Q}` — impossible by exhaustive enum of interaction patterns
2. Inhabit a `Carrier` not generated by `{F,O,S,P,T}` — impossible by free algebra completeness
3. Route to a `Seam` not in `{L,Cm,N,Cp}` — impossible by `LCNCMode` enum exhaustiveness
4. Combine factors in a way not expressible as `Use(Carrier) @ Seam` — impossible by cartesian product structure

Therefore no missing primitive exists. ∎

## 8. Mutual Exclusivity Proof

### 8.1 Statement

> **Theorem**: No two distinct taxonomy members denote the same signalling primitive.

### 8.2 Proof

We prove pairwise disjointness of each factor, then the product.

#### 8.2.1 Use Factor Disjointness

- `Observe` returns a value *synchronously*; `Subscribe` returns a *stream*
- `Command` takes a *mutator* and returns `Unit`; `Query` takes a *pure function* and returns a *value*
- `Query`'s irreducibility (§2.2) separates it from `Command × Subscribe`
- `Observe` is `Query(id)` but seam routing differs (§4.2) — kept distinct per architecture

#### 8.2.2 Carrier Factor Disjointness

Proven in §3.3 via normal form uniqueness.

#### 8.2.3 Seam Factor Disjointness

`LCNCMode` is an `enum class` with exactly 4 values. The `FacetEngine.dispatch` function is a *total* function from `FieldSynapse.opcode` to `LCNCMode`. Each opcode maps to exactly one mode. No overlap.

#### 8.2.4 Product Disjointness

The taxonomy is the cartesian product `Use × Carrier × Seam`. Since each factor is a set of pairwise-disjoint atoms, the product atoms `(u,c,s)` are pairwise distinct by definition of cartesian product. ∎
