A Cursor is just Join all the way down. Row indexing is the outer Join. Column indexing is the inner Join. Cell access is the innermost Join. No arrays. No maps. No managers. Just composed functions.

α composes two functions: the existing oracle b and the transform xform.

No collection is created. No iteration happens. The composition is deferred until access.

val .view

66inline class IterableSeries<T>(val series: Series<T>) : Iterable<T>, Series<T> by series {
67 override fun iterator(): Iterator<T> = object : Iterator<T> {
68 private var i = 0
69 override fun hasNext() = i < series.size
70 override fun next(): T = series[i++]
71 }
72}
74val <T> Series<T>.view: IterableSeries<T>
75 get() = this as? IterableSeries ?: IterableSeries(this)

.view is the only bridge from Series algebra to stdlib iterators. It's idempotent — calling .view.view returns the same IterableSeries.

This is the boundary where lazy composition ends and eager traversal begins. Use it for for loops, .map, .filter, .joinToString — stdlib territory.
